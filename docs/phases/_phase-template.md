# PHASE XX. 제목

이 문서는 VS Code Codex에 그대로 붙여넣어 실행할 수 있는 마음나무 구현 지시서 템플릿이다.

## 목표

- 이 Phase에서 달성할 구현 목표를 적는다.
- Phase 완료 후 사용자가 확인할 수 있는 산출물을 적는다.

## 작업 전 확인 사항

Codex는 작업 전에 반드시 아래를 수행한다.

```txt
1. 현재 코드베이스의 파일 구조를 확인한다.
2. docs/codex-common-rules.md를 읽는다.
3. 현재 Phase의 참조 문서를 읽는다.
4. 이미 구현된 파일과 충돌 가능성을 확인한다.
5. 현재 Phase 범위 밖의 기능은 구현하지 않는다.
```

## 참조 문서

```txt
docs/codex-common-rules.md
docs/...
```

## 수정 또는 생성할 파일

예시:

```txt
api/app/...
web/src/...
docker-compose.yml
.env.example
```

기존 파일이 있으면 삭제하지 말고 필요한 범위에서 수정한다.

## 구현 내용

### 1. 항목명

- 구현할 내용을 구체적으로 적는다.
- 함수, API, 컴포넌트, 상태값, 테스트 기준을 명확히 적는다.

### 2. 항목명

- Phase 범위 안에서만 구현한다.

## 금지 사항

```txt
- 기존 파일을 임의로 삭제하지 않는다.
- 현재 Phase 범위 밖의 기능을 구현하지 않는다.
- 기술 스택을 임의로 변경하지 않는다.
- 실제 API Key를 코드에 넣지 않는다.
- DB/API/세션 구조를 임의로 새로 만들지 않는다.
- LLM에 진단, 위험도 판단, 공개 여부 판단을 맡기지 않는다.
```

## 완료 기준

```txt
- 구현 대상 파일이 생성 또는 수정되어 있다.
- 관련 API 또는 화면이 동작한다.
- 기본 테스트가 통과한다.
- Docker 또는 로컬 실행 방법이 정리되어 있다.
- 남은 TODO가 명시되어 있다.
```

## 테스트 방법

```bash

cd api
pytest

cd web
npm run build
```
