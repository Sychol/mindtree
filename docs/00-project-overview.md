# 00. Project Overview

## 1. 서비스 정의

마음나무는 소방안전박람회 부스 방문자가 QR로 접속해 자신의 마음상태를 짧게 점검하고, 마음카드와 응원 문장을 남기면, 그 문장에서 추출·정제된 익명 키워드가 TV 화면의 나무 잎 형태 워드클라우드로 표시되는 이벤트형 리본톡 체험 서비스다.

이 서비스는 진단, 상담, 치료 서비스가 아니다. 박람회 현장에서 리본톡의 핵심 가치를 짧게 경험하게 하는 익명 참여형 마음 점검·응원 이벤트로 정의한다.

## 2. 핵심 사용자

### 2.1 참가자

부스 방문자다. 회원가입 없이 QR로 접속해 이벤트에 참여한다.

참가자가 수행하는 일은 다음이다.

```txt
QR 접속
→ 익명 세션 생성 또는 복구
→ 필수 동의
→ 1~77번 문항 응답
→ 마음신호 요약 확인
→ 마음카드 작성
→ 타인 카드 선택
→ 응원·공감·작은 대처법 작성
→ 완료 코드 확인
```

### 2.2 TV 관람자

부스 현장에 설치된 TV를 보는 사람이다. TV에서는 개인 결과나 원문이 아니라, 정제된 키워드와 참여 흐름만 본다.

TV에 표시되는 정보는 다음으로 제한한다.

```txt
- 전체 참여자 수
- 마음신호 키워드 TOP 5
- 응원·회복 키워드 TOP 5
- 나무 잎 형태 워드클라우드
- 체험형 이벤트 안내 문구
- TV에는 익명 키워드만 표시된다는 안내 문구
```

### 2.3 관리자

현장 운영자 또는 서비스 운영자다. 관리자 화면에서 검수, 키워드 상태, 완료 코드, 상품 지급 상태를 관리한다.

관리자가 수행하는 일은 다음이다.

```txt
- 이벤트 상태 확인
- 참여자 수와 완료율 확인
- 마음카드 검수
- 응원 문장 검수
- 위험 플래그 확인
- 키워드 상태 확인
- 키워드 숨김/수정/재계산
- 완료 코드 조회
- 상품 지급 처리
- 관리자 행위 로그 확인
```

## 3. 전체 운영 흐름

```txt
[참가자 모바일]
QR 접속
→ 이벤트 설정 조회
→ 익명 세션 생성/복구
→ 1~77번 문항 선로딩
→ 필수 동의
→ 문항 응답
→ 서버 점수화
→ 위험 플래그 계산
→ 마음신호 요약 확인
→ 마음카드 작성
→ 키워드 job 생성
→ 타인 카드 선택
→ 응원 문장 작성
→ 키워드 job 생성
→ 완료 코드 발급

[서버 내부]
마음카드/응원 저장
→ 안전 필터
→ keyword job 생성
→ LLM 또는 fallback 키워드 추출
→ 키워드 정제·유사어 병합
→ TV 집계 반영

[TV]
SSE 연결
→ 최신 키워드 집계 수신
→ 마음나무 워드클라우드 갱신
→ 연결 끊김 시 마지막 데이터 유지 및 자동 재연결

[관리자]
검수 대상 확인
→ 공개/숨김/수정/삭제
→ 키워드 상태 확인
→ 완료 코드 확인
→ 상품 지급 처리
→ 감사 로그 저장
```

## 4. 핵심 운영 원칙

## 4.1 참가자 흐름 우선

박람회 현장은 네트워크가 불안정할 수 있다. 따라서 참가자는 문항 로딩 후 서버 통신이 느려져도 최대한 계속 진행할 수 있어야 한다.

```txt
- 이벤트 진입 시 1~77번 문항을 한 번에 선로딩한다.
- 문항 이동은 프론트엔드 로컬 상태 기준으로 처리한다.
- 제출 실패 시 입력값을 유지하고 재시도 버튼을 제공한다.
- 서버는 같은 세션의 중복 제출을 idempotent하게 처리한다.
```

## 4.2 LLM 비차단 원칙

LLM은 보조 역할만 수행한다. LLM 응답이 늦거나 실패해도 참가자의 완료 흐름은 막히면 안 된다.

```txt
LLM이 할 수 있는 것:
- 마음신호 요약 문장 보정
- 마음카드 키워드 후보 추출
- 응원 문장 키워드 후보 추출
- 문장 톤 정리

LLM이 하면 안 되는 것:
- 진단
- 위험도 최종 판정
- 자살위험 최종 판단
- 공개 여부 결정
- 세션 상태 결정
- 관리자 승인 대체
```

## 4.3 공개 제한 원칙

공개 영역은 엄격히 제한한다.

```txt
개인 결과: 비공개
척도 점수: 비공개
위험 플래그: 비공개
마음카드 원문: 다른 참가자 카드 선택 화면에 한해 익명 공개 가능
응원 문장 원문: MVP에서는 TV 미표시, 관리자 검수 대상
TV: 익명 키워드만 표시
관리자: 운영 목적상 원문 확인·수정·숨김·삭제 가능
```

## 4.4 서버 판단 원칙

중요한 판단은 백엔드의 명시적 로직으로 처리한다.

```txt
- 세션 상태 전이
- 척도 점수화
- 위험 플래그 계산
- 공개 제한 여부
- 완료 코드 발급
- 상품 지급 상태 변경
- 관리자 감사 로그 기록
```

LLM은 이 판단을 대체하지 않는다.

## 5. 이벤트 완료 조건

참가자가 다음 조건을 만족하면 완료 코드가 발급된다.

```txt
동의 완료
+ 기본 문항/척도 응답 완료
+ 마음신호 요약 확인
+ 마음카드 1개 이상 작성
+ 타인 카드 선택
+ 응원·공감·작은 대처법 중 1개 작성
= 완료 코드 발급
```

완료 코드는 상품 지급 확인용이다. 이름, 전화번호, 소속 등 직접 식별정보는 기본적으로 수집하지 않는다.

## 6. 성공 기준

### 6.1 참가자 기준

```txt
- QR 접속 후 회원가입 없이 참여 가능하다.
- 문항 선로딩 후 네트워크가 느려져도 응답 진행이 가능하다.
- 제출 실패 시 입력값이 사라지지 않는다.
- 마음신호 요약을 오래 기다리지 않고 확인할 수 있다.
- 마음카드와 응원 문장을 작성할 수 있다.
- 키워드 추출을 기다리지 않고 완료 코드를 확인할 수 있다.
```

### 6.2 TV 기준

```txt
- TV에는 원문 없이 키워드만 표시된다.
- SSE 연결이 끊겨도 화면이 빈 상태가 되지 않는다.
- 자동 재연결이 가능하다.
- 키워드가 비동기로 반영되어도 자연스럽게 보인다.
- 참여자 수와 TOP 키워드가 주기적으로 갱신된다.
```

### 6.3 관리자 기준

```txt
- 부적절한 문장을 공개에서 제외할 수 있다.
- 위험 플래그 문장을 확인할 수 있다.
- 키워드 추출 실패 또는 대기 상태를 확인할 수 있다.
- 완료 코드로 상품 지급 여부를 확인할 수 있다.
- 중복 지급을 방지할 수 있다.
- 관리자 수정·삭제·지급 행위가 로그로 남는다.
```

### 6.4 안전 기준

```txt
- 위기 표현은 공개되지 않는다.
- 개인정보와 식별 가능한 사건 정보는 공개 제한된다.
- TV에는 개인 결과와 원문이 표시되지 않는다.
- LLM은 진단과 위험도 판단을 하지 않는다.
- LLM 실패가 서비스 실패로 이어지지 않는다.
```
